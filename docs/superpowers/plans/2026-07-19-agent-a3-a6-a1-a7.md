**Goal:** Ship the approved agent-expansion spec (`docs/superpowers/specs/2026-07-19-agent-a1-a6-a7-design.md`): sigma-runs scoping fix (A3), per-turn token metering (A6), propose→confirm agent annotations (A1), DB-backed admin agent config with reasoning effort (A7).

No new services. Writes stay analyst-executed: the agent only records proposals; a session-authenticated confirm endpoint performs the annotation write.

**Tech Stack:** Python 3.13, FastAPI, SQLAlchemy async + Alembic, pydantic-ai **2.13.0** (`RunUsage.input_tokens`/`output_tokens`; `OpenAIChatModelSettings.openai_reasoning_effort`; `AnthropicModelSettings.anthropic_thinking`; `ModelSettings.extra_body`), mcp `FastMCP`, React 19 + TanStack Query + Vitest.

## Global Constraints

- Ruff: `select = ["E","F","I","UP","B","C4","SIM"]`, line-length 100, E501 ignored — don't wrap for length. Google-style docstrings.
- Every `EventQueryService` call from an `async def` handler goes through `run_in_threadpool`.
- Migrations must run on SQLite too: `sa.func.now()`, no Postgres-only DDL. Create with `uv run alembic revision --autogenerate -m "..."` then hand-check.
- Token counts are **measured or NULL — never estimated** (spec, forensic honesty).
- The model never supplies case/timeline IDs to tools (scope-bound closures).
- `docs/AGENT.md` updated in the same commit as any `src/vestigo/agent/` change; `docs/ROADMAP.md` items deleted as they land.
- Test commands: `uv run pytest tests/<file>.py -v` (backend), `cd frontend && npm run test` + `npm run typecheck` (frontend).
- Commit after every task (messages given per task).

/** Compact token count: 890, 12.4k, 1.2M. */
export function formatTokenCount(n: number): string {
  if (n >= 1_000_000) return `${(n / 1_000_000).toFixed(1).replace(/\.0$/, "")}M`;

}

async def _resolve_event_sources(
    scope: AgentScope, event_ids: list[str]

    """Resolve event_id -> source_id within scope; also return unknown ids."""

    return found, [i for i in event_ids if i not in found]

                           user=Depends(get_current_user)) -> dict[str, Any]:
    conversation = await _require_conversation(case_id, conversation_id, user)
    store = get_store()

    decided = await store.decide_agent_proposal(proposal_id, status="confirmed",
                                                decided_by=user.username)

    # what still resolves, report the rest — the record stays truthful.
    scope = await build_scope(case_id, conversation.timeline_id, user)
    found, unknown = await _proposal_resolver()(scope, [e["event_id"] for e in decided.events])
    from vestigo.db.postgres import ANNOTATION_ORIGIN_AGENT
    from vestigo.ingestion.pipeline import generate_id  # or wherever generate_id lives — grep it
    rows = []
    for event in decided.events:
        if event["event_id"] not in found:
            continue
        for ann_type, content in (("tag", decided.tag), ("comment", decided.comment)):
            if content:
                rows.append({
                    "annotation_id": generate_id("ann"),
                    "case_id": case_id,
                    "source_id": event["source_id"],
                    "event_id": event["event_id"],
                    "annotation_type": ann_type,
                    "content": content,
                    "created_by": user.username,
                    "origin": ANNOTATION_ORIGIN_AGENT,
                })
    written = await store.bulk_create_annotations(rows)
    await store.record_audit(
        action="agent.annotation_confirm", actor=user, case_id=case_id,
        target_type="agent_proposal", target_id=proposal_id,
        detail={"conversation_id": conversation_id, "written": written,
                "skipped_event_ids": unknown, "tag": decided.tag,
                "comment_present": bool(decided.comment)},
    )
